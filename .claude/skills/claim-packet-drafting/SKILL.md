# Skill: claim-packet-drafting

## Purpose
Use this skill when building or improving AerisClaim claim packet drafting flows, evidence panels, claim detail views, and claim narrative generation logic.

## Goals
- Help create complete, structured claim packets
- Surface missing evidence clearly
- Support human review and editing
- Keep provenance visible

## Use this skill for
- claim detail page architecture
- packet completeness checks
- evidence checklist design
- draft generation prompts and flows
- follow-up and appeal draft flows

## Core principles
1. Claims must be auditable.
2. Every draft should map to source evidence.
3. Missing evidence must be explicit, not hidden.
4. Human review is a first-class part of the workflow.
5. Drafting UX should feel like document review, not chat.

## UI guidance
- Show source documents beside extracted facts and draft text
- Show evidence completeness clearly
- Show deadlines and submission readiness prominently
- Use diff/review patterns for edits
- Preserve version history for drafts
